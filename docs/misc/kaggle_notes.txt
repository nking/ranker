to run the code on kaggle:

(1) make the ranker-app kaggle upload:
    follow scripts/make_kaggle_upload.sh
(2) upload local data to kaggle notebook as input datasets
 - make tarballs of the 3 db container images and the main app image
    docker export $(docker create postgres:12-alpine) -o pg-db.tar
    docker export $(docker create fsouza/fake-gcs-server:1.54.0) -o fake-gcs-server.tar
    docker export $(docker create vizier-server:local) -o vizier-server.tar

    compress each of the tar archives
    then rename them to <name.tar.gz>.bin

    renaming them to *.bin bypasses a file filter check that fails
    for various harmless file name patterns.

make a TMP directory and cd into it
    mkdir TMP
    cd TMP

activate the venv you've made with the kaggle cli installed in it:
    e.g. conda activate kaggle_py313

kaggle datasets init -p .

for each dataset:

copy the <name.tar.gz>.bin file into TMP.  TMP can only have the file
to be uploaded and the dataset-metadata.json file in it.

If creating the dataset:
    kaggle datasets create -h

else if updating, 
    touch dataset-metadata.json

edit the file dataset-metadata.json file for the upload
    e.g.
{
  "title": "pg-db.tar",
  "id": "nicholeasuniquename/pg-db.tar",
  "licenses": [
    {
      "name": "CC0-1.0"
    }
  ]
}

for updates:

kaggle datasets version -p . -m "Your update message"

(3) prepare the datasets for upload
   cd to base of project
   cd tmp_mounts/fake-gcs-server/data && zip -r ../../../ml-ranker-dataset2.zip .
   cd ../../../
   mv ml-ranker-dataset2.zip TMP/
   edit dataset-metadata.json for the dataset
   add or update using kaggle cli

(2) install proot on kaggle

!apt-get update -y && apt-get install -y proot curl ca-certificates

export TMP=/tmp

then see the notebook for how to use:
https://www.kaggle.com/code/nicholeasuniquename/ranker-cross-encoder-w-gatv2/


(3) choose the T4x2 accelerator on kaggle

   GPU (Tesla T4/P100) : 16 GB RAN
   TPU v3-8             127 GB RAM
   TPU v5e-8            optimized for serving/train
                        has better performance per watt
                        and faster matrix ops than v3

   for compute bound: use TPU v5e-8 if possible
