Jax Coordination Service:
  - jax uses gRPC for its coordination service.
  - in a multi-container setup, the coordinator
    must be reachable at the exact IP and port
    specified in JAX_COORDINATOR_ADDRESS 
    from all workers.

Jax multi-node/multi-process across containers:
  - setup distributed communication
  - jax.distributed.initialize() on each host
    allows the processes to discover each other
    and coordinate device access
  - if using Docker, increase shared memory with 
    --ipc=host or --shm-size to prevent data 
    loader bottlenecks.

    across a cluster are local, that is 
    jax.process_index() will be 0 on all nodess.


Docker bridge:
  - docker0
  - running a container without the --network option connects it
    to the default bridge which is usually 172.17.0.1
  - containers attached to the default bridge can access 
    network services outside of the docker host
    (if host has internet access, so does container via
    'masquerading')
    -  e.g. docker run --rm -ti busybox ping -c1 docker.com
  - when containers use the default bridge network,they have
    access to each other using the container IP address.
  - to create groups of containers for which containers 
    have intra-group access but not inter-group access:
    -  ADD DETAILS HERE
  - docker drivers avail:
    - bridge:
      default network driver
    - host
      remove network isolation between the container and
      the Docker host
    - none
      completely isolate a container from the host and
      other containers
    - overlay
      swarm overlay networks connect multiple Docker
      daemons together
    - ipvlan
      connect containers to external VLANs
    - macvlan
      containers appear as devices on the host's
      network.
  - container can be connected to multiple Docker
    networks, such as a bridge netowrk and a --internal
    network

test if port is open from within a worker container:
  docker exec -it [worker_1_id] nc -zv 172.17.0.1 8888.
  docker exec -it [worker_1_id] ss -tulpn | grep 88
