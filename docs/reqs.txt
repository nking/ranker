goal is to make a ranker for the recommender systems
and MLOps pipelines for it.

Already exists:
- Data:
  - movie lens ratings, user and movie data
  - results from retrieval component of the system
    in the form of user query and resulting top_k recommended movies.

To be implemented in this project:
- trained Model 
- MLOps pipelines

Items not yet decided for MLOps:
  - the equivalent of MLMD for artifacts and models
    and how to incorporate that into kubeflow pipeline
    design.

Model requirements:
  Graph Transformer (GAT) as graph contextualized listwise ranker
  - must be trainable on tpu, gpu, and cpu
  - must be highly scalable
  - must be highly distributable
  - must run on linux OS at least
  - inference have fast response times (on the order of msec)
  - deployment should be highly available for online inference and
    have low latency.
  - must be capable of inference and bulk inference
    - given:
      a query of user ...<TBD>, unranked recommended movies
      returns ranked list of movies
  - trained model eval metrics ndcg@k should be an
    improvement over the un-ranked retrieval results
    and should be at least <TBD>.
  For this will use Jraphx + Flax NNX + Rax for creating the model
    and will save it to a JAX serving format

MLOps details:
  kubeflow pipelines
  the stars are highest priority
  - training and deployment pipeline:
    - * ingest data
    - calc statistics of data
    - generate data schema and perist it
    - validate data statistics
    - compare data to latest artifact
    - * transform data
    - * tune the model for best hyper-parameters
    - * train a model with best hyper-params
    - * evaluate model performance from the validation data
      metrics compared to baseline model.
      result is "BLESSED" or "NOT_BLESSED"
    - allow for approval
  - deployment pipeline
    - BLESSED and approved model can be deployed to serving
