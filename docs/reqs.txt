goal is to improve personalization in movie recommendations 
for the user.

this will be measured by NDCG@20 metric.

This is often accomplished by following the bi-encoder
+ Retrieval with a cross-encoder.

The cross-encoder (Ranker) in this project will use
a graph attention network with efficient, query-dependent
attention, a.k.a. GATv2, as the first layer in a
neural network, followed by a linear layer for scoring. 

GraphRanker as graph contextualized listwise ranker:
    GNN Cross Encoder (JraphX):
        Processes the local graph (user + candidates) to capture
        structural relationships (e.g., "movie_a is similar to movies user liked").
        jraphx has support for GATv2
    Listwise Interaction (Flax NNX):
        A small Transformer block that "compares" all candidates
        in the list simultaneously.
    Ranking Loss with (Rax)


The data needed by the model are a little complex so are
listed in detail here.
Data needed:
- train, val, test splits of the ratings:
  - 'liked' 
  - 'disliked' 
  - smaller sets of the above for use in development
- Retriever recommended unwatched movies for each user
- all movies
- files and runtime creation of negatives:
  - the natural hard negatives are the movies which the user rated 1 or 2
    see 'disliked' above
  - the "elite" hard negatives are the intersection of the natural hard 
    negatives with the recommended unwatched movies
  - the approximate hard negatives are the samples drawn from unwatched 
    movies during runtime
