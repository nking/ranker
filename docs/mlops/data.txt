- preprocessing
    - user and movie embeddings
      - created as outputs of bi-encoder in rec sys/tfx pipelines project
        - formats are array_record
    - user recommendations: ANN search results of movie_ids of movie vectors
          ranked by highest similarity to uer vector
      - created as outputs of bi-encoder in rec sys/tfx pipelines project
        - formats are array_record
    - ratings and negatives
      - created rec sys/tfx pipelines project
        - formats are array_record
    - movies catalog
      - created rec sys/tfx pipelines project
        - formats are array_record
- ingestion
    - input array records
      - grain dataloaders (backend fake gcs server, hosted by a container)
        - train dataloaders are configured to use SPMD across all jax
          processes.  the same model and hyperparameters (HP) are trained
          on data partitioned to the shard.
- tracking:
  - HPO (tune) 
    - metrics per trial and per epoch:
      - OSS Vizier server (hosted by a docker container)
      - MLFlow db (backend is postgres, hosted by a container)
  - train 
    - metrics per epoch:
      - MLFlow db (backend is postgres, hosted by a container)
    - latest 2 checkpoints
      - Orbax (backend fake gcs server, hosted by a container)
    - best checkpoint
      - Orbax (backend fake gcs server, hosted by a container)

======================
MLFlow:
  - experiment:
     - experiment_name: study_name
     - parent run: 
         - run_name: "tune"
         - each trial run: 
            - run_name: "trial_{trial_id}"
         - train run: 
            - run_name: "train_{train_id}"
         - test run: 
            - run_name: "test_{test_id}"
OR 
   if no tune phase preceeded a trial run
  - experiment:
     - experiment_name: study_name
     - parent run: 
         - run_name: "train"
         - train run: 
            - run_name: "train_{train_id}"
OR 
   if no tune phase preceeded a test run
  - experiment:
     - experiment_name: study_name
     - parent run: 
         - run_name: "test"
         - train run: 
            - run_name: "test_{test_id}"

xax.local_device_count():
    The number of accelerators attached to this specific 
    process/container.	
jax.device_count():
    The total number of accelerators across all 
    processes/workers in the cluster.
jax.process_count():
    the # of jax processes assoc with the backend.
    the number of hosts or containers in the cluster.

==> jax.device_count() = jax.process_count() * jax.local_device_count()
      if the device count is same across all nodes
